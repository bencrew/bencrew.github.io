    When: 2016
    Tech: Android Studio

    This Project was to implement a single tap game for use on a mobile device, the game was made using Android Studio. The game that was made was based on a group elevator pitch, the initial pitch was a single tap game whereby the user would have to get by waves on enemies that came towards the players ship at the bottom of the screen. The user was able to maneuver the ship along the screen to dodge and shoot the enemies. The game concept used memes as its basis and built a story around this.

    The mechanics of the game involve the player using the touch screen to move the ship left and right across the screen. The difficulty begins fairly simply with only one enemy on the screen at a time, but for every five enemies that you survive another will spawn. The objective is to survive as many waves as possible and achieve a high score. The game features a start screen, a menu screen with a brief story, settings menu where the in game music can be controlled.

    Game Engine

    Module: Games Programming

    When: 2016
    Tech: C++, openGL, XML

    This was a group project, where we had to make our own game engine and then create a game using the engine we had created. It was created from scratch using C++ and OpenGL. The engine the group created contained the following elements:

  • Object loading (FBX)
  • Texturing
  • Ambient lighting
  • Data driven XML loader
  • NPC AI
  • Collision physics
  • Sound manager
    • I worked on object loading through XML parsing using tiny XML, every object was loaded into the scene through XML and all of an objects data would be loaded along with the model such as height, width, length and data to do with the type of object loading i.e. health or ammo.
    I also wrote custom collision physics for the engine, this enabled the player to move around the world and interact with object within it.
    A demo reel for the final game can be seen here. Door Blasters

    When: 2016
    Tech: C#, Unity3D, Leap Motion

    My final year project was to create a fully functioning fruit machine that could be controlled by using a contact-less controller. I also wrote a dissertation piece titled "Fruit Machine using Leap Motion Controller" This looked at the evolution of the fruit machine and the possible innovation of the future.
    The Fruit machine was controlled my using a touch-less piece of equipment called the Leap Motion, it tracks hand motions in real time and allows the user to control buttons in a virtual environment.
    The fruit machine had the following features:

  • Checking for a win of 3 symbols on the center line
  • Spinning and randomised stopping of the reels
  • Chance of holding reels
  • Chance of bonus game on a win
  • Sound Effects
  • All buttons pressed using contact-less controller
